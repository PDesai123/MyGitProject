
-------------------------------------------
Test change tracking in local repository
-------------------------------------------

[1] Create a new file and save it to your local git repository folder

[2] open command prompt and run command "git status"

		D:\Cloud\VersionControl\GIT\Git-Repository>git status
		On branch master

		No commits yet

		Untracked files:
		  (use "git add <file>..." to include in what will be committed)
		        How To Create Local Repository.txt

		nothing added to commit but untracked files present (use "git add" to track)


[3] Add file to git repository index. Git will start tracking changes.

		D:\Cloud\VersionControl\GIT\Git-Repository>git add -A


[4] check status again after adding file


		D:\Cloud\VersionControl\GIT\Git-Repository>git status
		On branch master

		No commits yet

		Changes to be committed:
		  (use "git rm --cached <file>..." to unstage)
		        new file:   HowToCreateLocalRepository.txt

[4] commit changes to local repository using "git commit -m "<message string"


		D:\Cloud\VersionControl\GIT\Git-Repository>git commit -m "My first commit."
		[master (root-commit) 16f74c8] My first commit.
		 1 file changed, 19 insertions(+)
		create mode 100644 HowToCreateLocalRepository.txt

		D:\Cloud\VersionControl\GIT\Git-Repository>
